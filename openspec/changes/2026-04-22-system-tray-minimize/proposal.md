| 字段 | 值 |
|------|-----|

| **状态** | proposed |
| **创建日期** | 2026-04-22 |

| **关联变更** | 无 |

## 概述

为 mmpassword 添加系统托盘支持，实现关闭/最小化按钮隐藏到托盘，通过托盘图标菜单恢复窗口或退出应用。

## 目标

1. **关闭到托盘** — 点击关闭按钮不退出应用，而是隐藏到系统托盘
2. **最小化到托盘** — 点击最小化按钮同样隐藏到托盘
3. **托盘菜单** — 右键托盘图标弹出菜单（显示主界面 / 退出）
4. **托盘左键恢复** — 左键点击托盘图标直接显示并聚焦窗口

## 范围

### 包含功能

| 模块 | 功能 |
|------|------|
| **系统托盘** | 创建托盘图标，使用应用图标 |
| **关闭拦截** | 拦截 `CloseRequested` 事件，隐藏窗口而非关闭 |
| **最小化拦截** | 拦截 `Minimized` 事件，隐藏窗口而非最小化到任务栏 |
| **托盘菜单** | 右键菜单：显示主界面、退出 |
| **窗口恢复** | 左键点击 / 菜单"显示" → `window.show()` + `window.set_focus()` |

### 不包含

- 托盘图标通知气泡/提示
- 首次关闭到托盘的用户引导提示
- 最小化到系统任务栏的行为（改为全部到托盘）
- 托盘图标的动态变化（如未读数角标）

## 技术方案

### Rust 端实现（全部在 lib.rs setup 闭包中）

```
setup(|app| {
    1. 创建托盘图标 + 菜单
       - TrayIconBuilder::new(app)
       - icon: 应用图标 (icons/32x32.png)
       - tooltip: "mmpassword"
       - menu: [显示主界面, 分隔线, 退出]

    2. 注册托盘菜单点击事件
       - "show" → window.show() + set_focus()
       - "quit" → app.exit(0)

    3. 注册托盘左键点击事件
       - Click (Left, Up) → window.show() + set_focus()

    4. 拦截窗口关闭事件
       - on_window_event match CloseRequested
         → api.prevent_close() + window.hide()

    5. 拦截窗口最小化事件
       - on_window_event match Minimized
         → window.hide()
})
```

### 数据流

```
窗口事件                        托盘操作
─────────                      ─────────

CloseRequested                  左键/菜单"显示"
  │                               │
  ├─ prevent_close()              ├─ window.show()
  └─ window.hide()               └─ window.set_focus()

Minimized                       菜单"退出"
  │                               │
  └─ window.hide()               └─ app.exit(0)
```

## 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| `prevent_close()` 在某些 Tauri 版本有 bug | 中 | 确保使用最新 Tauri 2.x 版本，测试验证 |
| Linux 桌面环境托盘兼容性差异 | 中 | 主流环境 (GNOME + 扩展, KDE, XFCE) 测试覆盖 |
| 托盘图标尺寸不匹配 | 低 | 使用 32x32.png 作为托盘图标 |
| 应用无法通过常规方式退出 | 低 | 托盘菜单提供明确的"退出"选项 |

## 与现有功能的交互

| 功能 | 影响 | 处理方式 |
|------|------|----------|
| 自动锁定 (`useAutoLock`) | 无影响 | 继续按超时正常锁定，不受窗口隐藏/显示影响 |
| 前端 Screen 路由 | 无影响 | 窗口隐藏/显示不改变前端状态 |
| Git 同步 | 无影响 | 后台同步不受窗口可见性影响 |
